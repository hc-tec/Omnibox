“全景式”私有数据源地图
1. 创作与反馈平台 (用户的“公开资产”)
● Bilibili (B站): 投稿列表、收到的评论/弹幕、收藏夹、点赞/投币记录、后台数据 (完播率、粉丝画像)。
● 知乎 (Zhihu): 发布的文章/回答、收藏夹、赞同/喜欢、收到的评论。
● 小红书 (Xiaohongshu): 发布的笔记、收藏夹、收到的评论/私信。
● 抖音 / 视频号: 发布的作品、收藏/喜欢、收到的评论。
2. 知识与沉淀工具 (用户的“第二大脑”)
● 笔记工具 (语雀, Notion, Obsidian等): 所有文档/页面、剪藏内容、数据库 (Database)。
● 稍后读 (Cubox, Instapaper): 收藏的文章列表、归档。
● 云盘 (百度网盘, 阿里云盘, Google Drive): 特定授权的文件夹 (如“我的 PDF 库”、“项目素材”)。
● 文献管理 (Zotero, Mendeley): 论文库、文献元数据、笔记。
3. 专业与技能社区 (用户的“技术栈”)
● GitHub: Starred 仓库、自己的仓库/提交、关注的 Issues/PRs。
● 技术社区 (掘金, CSDN, V2EX): 收藏/点赞的文章、发布的文章、关注的标签。
4. 社交与信息流 (用户的“隐性兴趣”)
● 微博 / X (Twitter): 收藏/喜欢 (Likes)、关注列表、“特别关注”的动态。
● 即时通讯 (微信, 飞书):微信收藏 (高价值、零散)、飞书云文档 (工作思考)、授权的群聊记录。

5. 工作与效率工具 (用户的“执行队列”)
● 电子邮件 (Gmail, Outlook):
  ○ 收件箱/特定标签: (如“Newsletter”、“账单”、“工作简报”)。
  ○ 已发送 (Sent): 用户的“对外沟通”记录。
  ○ 星标邮件 (Starred): 用户标记的“高优”信息。
● 日历 (Google Calendar, 飞书日历, Outlook Calendar):
  ○ 日程安排 (Events): 用户的“时间分配”和“会议主题”。
  ○ 重复性任务: 用户的“固定习惯”。
● 任务管理 (Todoist, TickTick, Trello, Asana):
  ○ 项目列表 (Projects): 用户的“核心目标”。
  ○ 已完成的任务 (Completed): 用户的“产出记录”。
  ○ 收集箱 (Inbox): 用户的“待办”思绪。
6. 浏览与本地文件 (用户的“真实沙盒”)
● 浏览器 (Chrome, Edge, Safari):
  ○ 书签栏 (Bookmarks): 用户的“常用”和“永久收藏”的工具/网站。
  ○ 历史记录 (History): （需用户授权）用户的“原始研究轨迹”。
● 本地文件夹 (Local Folders):
  ○ 下载 (Downloads): 用户的“临时”文件和“素材”。
  ○ 桌面 (Desktop): 用户的“高频”和“正在处理”的文件。
  ○ 特定项目文件夹: （用户主动添加）如 C:\My_Projects\ 或 ~/Documents/Work/。
7. 消费与学习平台 (用户的“具象化需求”)
● 电子阅读 (微信读书, Kindle):
  ○ 高亮/划线 (Highlights): 用户认为的“知识重点”。
  ○ 读书笔记 (Notes): 用户的“即时思考”。
  ○ 书架列表 (Library): 用户的“认知边界”和“兴趣图谱”。
● 播客 (Apple Podcasts, 小宇宙, Pocket Casts):
  ○ 订阅列表 (Subscriptions): 用户的“被动学习”渠道。
  ○ 收藏的单集 (Saved Episodes): 用户的“待听”和“高价值”单集。
8. 商业与意图数据 (用户的“消费画像”)
● 电商平台 (淘宝, 京东, Amazon):
  ○ 购物车 (Shopping Cart): 用户的“短期”和“明确”的购买意图。
  ○ 收藏夹 (Wishlist): 用户的“长期”和“潜在”的兴趣。
  ○ 购买记录 (Order History): 用户的“真实偏好”和“消费能力”。
